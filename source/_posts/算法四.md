# 基础

本书的目的是研究重要而实用的`算法`，即适合用计算机实现的解决问题的方法，和算法关系最紧密的是`数据结构`，即便于算法操作的组织数据的方法。

首先介绍的是`基础编程模型`，书中用到了java语言中的一部分（我会尽量转换成JavaScript）。

## 算法

编写一段计算机程序一般都是实现一种已有的`方法`来解决某个问题，这种方法大多与使用的编程语言无关-它试用各种计算机和编程语言。是这种方法而非计算机程序描述解决问题的步骤，在计算机科学领域，用`算法`这个词描述一种优先、确定、有效的并适合计算机程序来实现的解决问题的方法，算法是计算机科学的基础，是这个领域的核心。

在本书中观点是`数据结构`是算法的副产品或者是结果，因此要理解算法必须学习数据结构，简单的算法也会产生复杂的数据结构，相应的复杂的算法也许只需要简单的数据结构，本书中将会探讨许多数据结构的性质，也许本书就应该叫做`算法与数据结构`。

本书共计六章

1. 基础

他讲解了随后章节中用来实现、分析比较算法的基本原则和方法，包括java编程模型、数据抽象、基本数据结构、集合类的抽象数据结构类型、算法性能分析的方法和一个案例的分析。
2. 排序

有序的重新排列数组中的元素是非常重要的基础算法，会深入讨论各种排序算法，包括插入排序、选择排序、希尔排序、快速排序、归并排序和堆排序。另外会讨论一些算法，他们用于解决几个与排序相关的问题，例如优先列队，选举及归并。
3. 查找

从庞大的数据集合中找到指定的条目也是非常重要的，将讨论基本和高级的查找算法，包括二叉查找树，平衡查找树和散列表。
4. 图

图的主要内容是对象和他们的链接，链接可能有权重和方向，利用图可以为大量重要而困难的问题建模，因此图算法的设计也是本书中一个研究领域，会研究深度优先搜索、广度优先搜索、连通性问题以及若干其他算法和应用。
5. 字符串

字符串是现代应用程序中最重要的数据类型，将会研究一系列处理字符串的算法，首先是对字符串键的排序和查找的快速算法，然后是子字符串查找，正则表达式匹配和数据压缩算法。
6. 背景

这一章将会讨论本书中有关其他前言研究领域，包括计算机科学、运筹学和计算理论，等等。

// 这段代码是经典的二分查找
class BinarySearch {

    rank (key, array) {
        let lo = 0,
            hi = array.length - 1
        while(lo <= hi) {
            const mid = Math.floor(lo + (hi - lo) / 2)
            if (key < array[mid]) {
                hi = mid - 1
            } else if (key > array[mid]) {
                lo = mid + 1
            } else {
                return mid
            }
        }
        return -1
    }

    search (key, array) {
        return this.rank(key,array)
    }
}

// 这段代码很简单就不写注释了，注意二分查找是查找有序的数组的

## 算法分析

随着使用计算机经验的增长，人美在使用计算机解决困难问题或者处理大量数据时不可避免的会产生疑问：

`我的程序会运行多长时间`，`为什么我的程序耗尽了所有内存`

这个过程的基础是`科学方法`，它是科学家为获取自然界知识使用的一系列为大家所认同的方法，将会使用`数学分析`为算法成本建立简介的模型并使用`实验数据`验证这些模型

### 科学方法

科学家用理解自然世界的方法对于研究计算机程序的运行时间同样有效：

1. 细致的`观察`真实世界的特点，通常还要有精确地测量
2. 根据观察结果预测未来的事件
3. 根据模型`预测`未来的事件
4. 继续观察并`核实`预测的准确性
5. 如此反复直到确认预测和观察一致

科学方法的一条关键原则是设计的实验必须是`可重现`的，这样他人也可以验证假设的可行性，所有的假设必须是`可证伪`的，这样才能确认某个假设是错误的。

### 观察

学习如何定量测量程序的运行时间，这个问题可以解决一个核心问题，我的程序会运行多长时间。

```JavaScript

class ThreeSum {
    count (array) {
        const N = array.length
        let cnt = 0
        for (let i = 0; i < N; i++) {
            for (let j = i+1; j < N; j++) {
                for (let k = j + 1;k < N; k++) {
                    if (array[i] + array[j] + array[k] === 0) {
                        cnt++
                    }
                }
            }
        }
        return cnt
    }
}
console.time(2)
const threeSum = new ThreeSum()
console.log(threeSum.count([1,2,3,4,5,6,7,8,-2,-1]))
console.timeEnd(2) // 0.49ms
```

### 数学模型

在计算机科学的早期，学者认为，尽管有许多复杂的因素影响我们对程序执行时间的理解，原则上，我们仍能构造出一个数学模型描述任意程序的运行时间，一个程序的总时间主要有两点有关：

1. 执行每条语句的耗时

2. 执行每条语句的频率

对于大多数程序，得到其运行时间的数学模型所需的步骤如下：

1. 确定`输入模型`定义问题的规模

2. 识别`内循环`

3. 根据内循环中的操作确定`成本模型`

4. 对于给定的输入，判断这些操作的执行频率，这可能需要进行数学分析

## 增长数量级的分类

描述|增长的数量级|典型代码|说明|举例
|--|----------|-------|---|---|
常数级别|1|a = b + c|普通语句|将两个数字相加|
对数级别|log N| 二分查找| 二分策略|二分查找|
线性级别|N|const a = 1; for(let i = 0; i < a; i++)|循环|找出最大元素|
线性对数级别|N log N|并归排序|分治|并归排序|
平方界别|N²|嵌套两层for循环|双层嵌套|检查所有元素对|
立方级别|Nm³|嵌套三层for循环|三层循环|检查所有三元组|
指数级别|2的N次方|穷举|穷举查找|检查所有子集|

常数级别

运行时间的增长数量级为`常数`的程序完成了任务所需的操作次数一定，因此它的运行时间比依赖于N

对数级别

运行时间的增长数量级为对数的程序比常数时间的程序稍慢，运行时间和问题和问题规模成对数关系，最经典的例子就是二分查找，对数的底数和增长的数量级无关，所以说明对数级别时一般使用log N

线性级别

使用常数时间处理输入数据中的所有元素或是基于单个for循环的程序十分常见，此类程序的增长数量级是线性的-它的运行时间和N成正比

线性对数级别

我们用先行对数描述运行时间和问题规模N的关系为NlogN的程序，和之前一样，对数的底数和增长的数量级无关，线性对数算法的最经典例子是sort排序，谷歌使用快速排序，火狐使用并归排序，两个算法的复杂度是一样的。

平方级别

平方级别的程序一般都含有两个嵌套的for循环，对于N个元素对进行计算。

立方级别

立方级别的程序一般含有三个for循环，对于N个元素得到的所有三元组进行计算

指数级别

指数级别的算法非常慢，不可能用他们解决大规模的问题，但是指数级别的算法任然在算法理论中有着重要的地位，因为他们看起来仍然是解决许多问题的最佳方案。

以上是`常见的分类`但是肯定不是最齐全的。

## 注意事项

对成簇的性能进行仔细分析时候，得到的不一致或是右误导性的结果的可能右很多种，他们都是由于我们的猜想基于一个或者多个假设并不完全正确造成的，我们可以根据新的假设得出新的猜想，但是我们考虑的细节越多，在分析中需要注意的方面也就越多

### 大常数

在首项近似中，一般会忽略低级项中的常数系数，但这可能是错的，我们要对可能的大常数保持敏感。

### 非决定性的内循环

内循环是决定性因素的假设并不总是正确的，错误的成本模型可能无法得到真正的内循环，有些程序在内循环之外也有大量指令需要考虑，换句话就是说，成本模型可能还需要改进

### 指令时间

每条指令执行所需的时间总是相同的假设并不总是正确的，由于缓存，对于大数组该比例可能突然变为很大的值

### 系统因素

一般来说计算机中总是运行着许多程序，这种因素可能会干扰到`实验必须是可重现的`这条科学院就的基本原则，原则来说此时计算机系统中运行的其他程序应该是可以忽略或者可以控制的

### 不分伯仲

在比较执行形同任务的两个程序时，常常出现情是其中一个在某种场景中更快而在另一些场景中更慢，对于这些从差异中找到最佳的实现，此类工作最好还是留给专家

### 对输入的强烈依赖

在研究程序的运行时间增长数量级时候，首先做出几个假设之一就是运行时间应该和输入有关，当这个条件无法满足的时候，很可能无法得到一致的结果或是验证猜想

### 多个问题参量

过去的重点一直是使用仅需要一个参量的函数来衡量程序的性能，参量一般是命令行参数或者是输入规模，但是多个参量是可能的
